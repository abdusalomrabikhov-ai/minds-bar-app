## Priority Issues

**[P0] Burgundy used as decorative KPI-card fill, breaking the One Accent Rule**
Why it matters: The "Выполнено" card is the single most prominent object on the dashboard, using the brand's one reserved action-color as pure background decoration — and it visually competes with the adjacent red "Просрочено" card, muddying "burgundy = do this" vs "red = actual problem" right at the point where that distinction matters most.
Fix: Restyle to match its three sibling cards — white surface, colored icon/value only, using --success (green) since 48% done is a positive metric, not --primary.
Suggested command: /impeccable polish (dashboard stat cards)

**[P0] Decorative left-border stripes on overdue task cards, outside the one permitted use**
Why it matters: DESIGN.md reserves this pattern exclusively for the sidebar's active-nav indicator so it stays a uniquely recognizable signal. Reusing it on task cards both breaks the rule and dilutes that signal everywhere else it appears.
Fix: Drop the border; the existing red "Просрочено: 7д 19ч" pill plus the section's red-tinted banner already carry the signal without redundancy.
Suggested command: /impeccable polish (overdue task list)

**[P1] ~25 undocumented hex values and 10 off-scale radii have drifted into the codebase**
Why it matters: Confirmed by direct source inspection (not a detector false positive) — a second green (#22c55e), a second amber (#ca8a04), and an indigo (#6366f1) compete with the documented semantic palette; radii like 4px/8px/6px sit off the documented 7/10/16/20px scale. This is exactly the kind of silent token drift DESIGN.md exists to prevent, concentrated outside the dashboard (task-detail/checklist UI per the grep hits) but visible in the same audit.
Fix: Either fold these into the documented token set (if intentional) or replace with existing tokens (if drift). Needs a decision pass before /impeccable extract can consolidate.
Suggested command: /impeccable audit (full color/radius token pass), then /impeccable extract

**[P1] Badge/pill proliferation makes each overdue task row take real effort to parse**
Why it matters: A single row shows priority, project, type, overdue-duration, status, and (per P0 above) a left-border — six simultaneous color signals, directly contradicting PRODUCT.md's own stated priority that density must serve readability, not fight it.
Fix: Cap at 2 color-carrying signals per row (priority + overdue status); demote project/type to plain-text or monochrome-outline chips.
Suggested command: /impeccable distill (task card component)

**[P2] Two independently-scoped period filters on one screen with no visible relationship**
Why it matters: The month carousel at the top and the separate Неделя/Месяц/3мес/6мес/1год filter lower down both answer "what time window," but aren't visibly linked — live testing surfaced a jarring case where "Нет активных задач за месяц" rendered directly beneath "27 просрочено," reading as contradictory rather than intentional.
Fix: Merge into one page-scoping control, or add a one-line label clarifying overdue items are period-independent.
Suggested command: /impeccable clarify (dashboard period filters)
